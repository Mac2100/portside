⬆ Git Deploy
A read-only GitHub token lets Portside pull your app repos onto the NAS and restart the container. It's shared by every app and stored encrypted in your Mac Keychain — never written to the NAS.
Create one at GitHub → Settings → Developer settings → Fine-grained tokens, with Contents: Read-only on your app repos.
ℹ️ About Portside
v—
Portside checks for new releases daily. Updates are downloaded from GitHub as a fresh DMG.
🐳 Image Updates
Watches the registries your containers pull from (Docker Hub, GHCR, lscr.io, …) for new versions of the images they run — like Watchtower.
Auto-update containers
Checked containers are pulled & recreated automatically when a check finds a new image. Unchecked ones just show an entry under Insights → Updates.
🐙 GitHub Watch
Watches source-code repos for new commits and releases. Any repo you configure in Git Deploy is watched and linked to its container automatically — no need to add it here. Runs on its own schedule, independent of Image Updates. Private repos use your Git Deploy token. The box below is only for extra repos you want to watch without deploying (e.g. upstream projects).
Per repo, pick how a push is handled: Notify = alert + one-click Deploy · Auto = deploy immediately · Nightly = queue it and deploy at the time above (for production apps). Portside must be running (menu bar is enough) and the Mac awake at that time.
